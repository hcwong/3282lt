Service workers are NOT offline helpers, NOT push notifications, NOT web workers

They are a worker that runs in the background

Running in the background gives service workers powerful features:

1. Intercepting and handling network requests

2. Many more features that do not need a web page or user interaction

3. In line with current web best practices

Key pointsExplanations
Why use service workers? Mobile apps have terrible adoption rate!
Service workers are a gateway drug There are no barriers to entry. Once you have hooked your user, then make them use the mobile app
Service workers offer an app-like experience It offers nice offline experiences when both viewing an interacting with the app, making it different from normal web pages
But there are pitfalls It's hard to write and test, though with more people using this, things will improve!
